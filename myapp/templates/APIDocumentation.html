Overview

The SMS Message Classification API is designed to classify SMS messages as either "Smishing" (fraudulent) or "Legitimate" based on their content. It uses a pre-trained machine learning model to make predictions. This documentation provides details on how to use the API, including endpoints, request and response formats, and examples.

Endpoint

'/detect'(POST):
This endpoint allows you to detect whether an SMS message is "smishing" or "Legitimate". The endpoint is by the following url: http://smishing-detector.onrender.com/detect

Request

Method: 'POST'

URL: http://smishing-detector.onrender.com/detect

Headers:
  Content-Type: 'application/json'

Request Body:
  JSON Object: Contains a single field:    'sms_message'(String required): The text of th SMS you want to analyze

Example Request:
  POST http://smishing-detector.onrender.com/detect
 Content-Type: application/json

 {
 'sms_message': "This is a sample SMS message to be analyzed."
 }

Response

Status Codes:
  '200 OK': The request was successful, and the analysis result is provided
  '400 Bad Request': The request is missing the required 'sms_message' field
  '500 Internal Server Error': An error occurred during processing

Example Response:
  HTTP/1.1 200 OK
 Content-Type: application/json

 {
 'result': "Legitimate"
 }

Usage

  1. Send a POST request to the '/detect' endpoint with a JSON payload containing the 'sms_message' field.
  2. The API will preprocess the SMS message, vectorize it using a TF-IDF vectorizer, and make predictions using the pre-trained machine learning model.
  3. The result, indicating whether the SMS message is "Smishing" or "Legitimate", will be returned in response.
  4. The SMS message and the result will be stored in the database for further analysis and auditing.

Error Handling

  • If the request does not contain the required 'sms_message' field, a '400 Bad Request' response with an error message is returned.
  • If an Internal server error occurs during processing, a '500 Internal Server Error' response is returned./li>

Notes

  • Ensure that your application is running and listening on the specified endpoint before making requests to API.
